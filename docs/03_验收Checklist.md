# 第一关交付 Checklist（纯 Web 3D）

> 全部勾完才视为“第一关达到一线产品原型质量”。验收分为资产、工程、玩法、浏览器与品质五部分。
> 几何占位、只有构建成功、只在单台桌面机器可见，都不能替代正式验收。

---

## A. 资产验收

### A1 统一骨骼与动画

- [ ] 共享骨骼层级、joint 命名、bind pose 和朝向统一。
- [ ] Idle / Run / Walk / LookAround / Scared / Celebrate / Point 动画齐全。
- [ ] 30 fps 烘焙，Root Motion 关闭，无滑步、抖动或穿模。
- [ ] 小孩与坏人的三套真实模型均能在 Three.js `AnimationMixer` 中复用动画。
- [ ] 带骨骼模型克隆后动画互不串扰，不出现角色消失或骨骼塌陷。

### A2 角色（小孩 / 坏人 / 警察 × 3 风格）

- [ ] 每套正式 GLB 通过 glTF Validator，Error=0；Warning 已解释。
- [ ] `GLTFLoader` 加载无异常，scale=1、面朝 +Z、轴心在脚底。
- [ ] PBR Metallic-Roughness 通道正确，WebGL 中无全黑、过曝或法线翻转。
- [ ] 面数、贴图分辨率、材质数、GLB 体积在预算内，附报告。
- [ ] 三风格共用骨骼契约，运行时切换无缝，逻辑状态和世界位置不丢失。
- [ ] 写实风格 LOD0/1/2 齐全，切换无明显跳变。
- [ ] 全真实 PBR 质感，无基础几何和纯色色块充数。
- [ ] 小孩、坏人、警察在俯视相机和手机屏幕上都清楚可见、一眼可辨。

### A3 环境（模块套件 + 道具 + 示例关卡）

- [ ] 模块 GLB 按 2 米网格对齐，拼接无缝、无 Z-fighting。
- [ ] 地面/墙面平铺材质无接缝，重复对象复用 geometry/material 或实例化。
- [ ] 前门与后门（警察局）地标醒目、方向可读。
- [ ] 警车红蓝灯和警局门厅 Emission 正常，移动端不过曝。
- [ ] `level.json` 中起点、终点、巡逻点、碰撞和导航数据完整。
- [ ] 从起点到终点可达；玩家与 AI 无穿墙、卡死或错误捷径。

### A4 资产目录与授权

- [ ] 当前可编辑母版和来源记录只在 `art-source/`；历史候选、拒稿、备份和生成预览不进仓库。
- [ ] 浏览器只加载 `public/models/` 下的 GLB、贴图和关卡数据。
- [ ] 产品代码不包含绝对本地路径、临时目录或 `art-source/` URL。
- [ ] 第三方素材来源、许可证和允许的使用范围有记录。
- [ ] 运行时资源请求无 404，MIME 类型和缓存策略正确。

---

## B. Web 工程与集成

- [ ] 根目录存在 `app/`、`public/`、`package.json` 与 `.openai/hosting.json`，不存在第二套产品入口。
- [ ] Three.js + WebGL 2 渲染路径就绪；不依赖任何本地桌面游戏引擎。
- [ ] `npm install` 后可直接 `npm run dev`，无未说明的手工步骤。
- [ ] `npm run lint`、`npm run build`、`npm test` 全部通过。
- [ ] 玩法逻辑不依赖具体 Mesh/材质；换模型不需要改移动、AI 或胜负代码。
- [ ] 角色资源 manifest 和风格配置集中管理，三风格可在运行时切换。
- [ ] 三套 WebGL 光照/后处理档接好，桌面和手机切换不违和。
- [ ] 页面卸载、重开和换风格时释放废弃 geometry、material、texture 和 mixer。
- [ ] 生产构建可由根目录托管配置发布，刷新游戏 URL 不出现路由错误。

---

## C. 玩法（第一关完整可玩）

### C1 移动与输入

- [ ] 桌面浏览器：方向键 / WASD 八方向移动，对角线归一化，转身和减速自然。
- [ ] 手机浏览器：屏幕方向控制可持续按压，松手立即停止，无页面误滚动。
- [ ] 点击寻路可重新指定目标；不可达点吸附到最近可行走位置。
- [ ] 键盘、触控按钮和点击寻路共用同一 `MoveIntent` / `PlayerMotor`。
- [ ] 窗口缩放、横竖屏切换和失焦恢复后不会卡键或持续移动。

### C2 反派 AI

- [ ] FSM 的 Idle → Chase → Search → Patrol 状态切换正确。
- [ ] 视锥、近身感知和遮挡判断生效；坏人不读取玩家全局位置作弊。
- [ ] 玩家点击寻路与 AI 共用同一可行走网格/导航图。
- [ ] 速度与转向平衡正确：坏人略慢、转弯更笨，玩家可通过路线选择甩脱。
- [ ] 感知半径、视锥角、速度系数、搜索时间和启动延迟由 TypeScript/JSON 配置控制。

### C3 关卡流程

- [ ] 起点前门、终点警察局区域生效。
- [ ] 胜利：进入警察局 → 警察演出 + 坏人退场 + 胜利面板。
- [ ] 失败：被抓 → 淡出 → 约 0.8 秒后重开。
- [ ] 首次追逐有约 1.5 秒反应时间。
- [ ] 连续胜利、失败和重开 20 次，无状态泄漏、重复角色或持续内存增长。

### C4 相机、表现与音频

- [ ] 相机俯视跟随、边界钳制，无穿墙露馅或角色移出画面。
- [ ] 玩家、追捕者和前方道路共同参与动态视野计算；变化平滑、不晕眩。
- [ ] 滚轮可调视野，`0` 恢复默认倍率；手机端自动视野足够看清局势。
- [ ] 坏人靠近时屏幕边缘泛红、心跳渐强，但不遮挡中心操作区。
- [ ] 自适应音乐按距离混音，脚步、警笛与胜负 Stinger 到位。
- [ ] 浏览器获得首次用户手势后正确恢复音频，无自动播放策略报错。

## D. 浏览器矩阵与响应式验收

- [ ] macOS Chrome 当前版完整跑通第一关。
- [ ] macOS Safari 当前版完整跑通第一关。
- [ ] iPhone Safari 真机完整跑通，触控、音频和安全区正确。
- [ ] Android Chrome 真机或等效测试设备完整跑通。
- [ ] 常见窄屏、宽屏、横屏和竖屏下，角色、HUD 和方向控制互不遮挡。
- [ ] UI 使用 `env(safe-area-inset-*)`，按钮触控热区 ≥44 CSS px。
- [ ] Canvas 像素比有上限；高 DPR 手机不会因过度分辨率明显掉帧或过热。
- [ ] 三套风格各完整跑一局：视觉自洽、无丢材质、无角色消失、无明显掉帧。

---

## E. 性能、加载与稳定性

- [ ] 主流桌面稳定 60 FPS；主流手机目标 60 FPS，中低端不低于 30 FPS。
- [ ] 常态 Draw Call <150；重复墙体、地面和道具正确复用/实例化。
- [ ] 首关关键资源压缩后目标 <60 MB；非当前风格按需加载。
- [ ] 可缓存运行时资源目标总量 <300 MB；制作源不进入生产构建。
- [ ] 目标网络下启动到可玩 <5 秒，加载进度真实、不假死。
- [ ] 控制台无未处理异常、WebGL warning 洪泛或资源 404。
- [ ] 重开、换风格和页面重新进入后内存回落，无明显 GPU 资源泄漏。
- [ ] 弱网下显示可恢复错误；单个非关键模型失败不导致白屏。

---

## F. 建议推进节奏

1. **先跑 Web 白盒**：在根应用中验证移动、AI、胜负和动态相机，但白盒不算最终交付。
2. **资产并行**：按 `02_Codex外包资产规格.md` 完成制作源、GLB 和报告。
3. **正式接入**：把运行时资产放入 `public/models/`，通过 manifest 接入并删除最终画面中的占位模型。
4. **浏览器打磨**：逐项完成动画、材质、相机、响应式、加载和内存优化。
5. **矩阵签收**：桌面 Chrome/Safari、iPhone Safari、Android Chrome 各跑一遍再签收。
