# 《追逐 / Chasing》Web 版深度打磨执行与回归方案

> 执行版本：2026-07-23
> 上位设计：07《Web 版统一优化方案与路线图》、08《躲藏机制与顶级动画执行规格》
> 原则：不以几何占位、删美术或降低角色辨识度换取性能。
> 当前状态：本轮 P0→P2 已落地并完成生产构建、自动回归、十关截图与桌面/移动端运行态验收。

## 1. 本轮目标

本轮不是增加一组彼此孤立的功能，而是把移动、躲藏、追捕、主题事件、声音、成绩与加载整合成同一个可验证闭环：

1. 屏幕方向与固定镜头保持永久一致；手机由四向按钮升级为连续虚拟摇杆。
2. 躲柜对齐阶段允许快速取消，正式进柜动作开始后保持动作承诺，避免穿模和状态反复。
3. 追捕者失去视线后必须到达最后目击点并左右巡视，之后才可调查新鲜声音或进入附近搜索。
4. 校园、医院、消防站、工厂各有周期性环境机制，并同时作用于画面、音频和合法 AI 感知。
5. 角色脚步由动画接触点驱动；柜门、衣料、撞击和结算使用正式录音素材，合成声仅作解码失败回退。
6. 十关至少存在两条可执行的藏身生还选择；标准引导首通不能大面积自动获得最高评价。
7. 首屏资源加载具备取消、超时、并发限制与指数退避；纹理压缩必须保留节点、骨骼、动画、材质和命名契约。
8. 四类主题机关从“定时背景效果”升级为场景内可接近、可触发、有预警、有代价、有冷却的主动博弈工具。
9. AI 只消费最多三条合法证据；重复滥用同一诱饵会降低可信度，搜索目标只能来自真实可达岔路。
10. 标准与辅助通关分轨保存，精通目标、个人最佳和压缩输入 Ghost 共同支撑复玩。

## 2. 玩法与 AI 执行合同

### 2.1 证据优先级

| 当前证据 | 新证据 | 处理 |
| --- | --- | --- |
| 无 | 目击 | 进入确认/追逐，记录精确最后目击点 |
| 无 | 声音 | 只记录带误差的声音位置 |
| 正在追逐目击点 | 声音 | 保存为次级声音证据，不覆盖视觉锚点 |
| 抵达目击点 | 无 | 完成完整左右巡视 |
| 巡视完成 | 新鲜且足够强的声音 | 调查次级声音证据 |
| 巡视完成 | 无有效声音 | 进入证据排序的附近搜索 |

追捕逻辑不得读取玩家当前隐藏坐标或柜内占用状态。柜子是否有人只能在完整搜柜动作结算时揭示。

AI 证据账本最多保留三条有效证据。视觉、目击进柜、环境声源和玩家声音各自携带来源、置信度与衰减；任何主题机关声源都必须先经过与玩家声音相同的合法听觉采样，再进入 AI。重复稳定声源会发生习惯化，可信度降到阈值后不再重置路径。环境证据触发的搜索假设固定生成 3–5 个真实导航岔路，禁止围绕隐藏玩家坐标造点。

### 2.2 躲藏交互

- `free → aligning-hide`：玩家开始靠近并转向柜门。
- `aligning-hide → free`：移动或再次互动可在 100 ms 内取消。
- `aligning-hide → entering-hide`：到达位置并完成朝向后提交正式动作。
- `entering-hide` 之后不接受取消，门、角色、暴露窗口和声音使用同一动作时序。
- 柜内观察继续使用按住控制；打开门缝后才恢复视觉暴露，松开后先关缝再允许离柜。

### 2.3 玩家公开情报

HUD 只消费公开威胁等级、可见追捕者表演和已经听到的宽泛音乐强度。不可见追捕者的精确模式、坐标和连续距离不得进入方向声、HUD 文案或 DOM class。

## 3. 四类主题机制

每个主题保留连续包络的环境周期，同时增加一个放置在关卡中的主动机关。主动机关采用 `ready → warning → active → cooldown → ready` 状态机；触发必须满足交互半径，激活时同时结算明示代价、正式动画/音频、空间作用区与合法 AI 声音证据。

| 主题 | 主动机关 | 玩法作用 | 表现要求 |
| --- | --- | --- | --- |
| 校园 | 走廊铃声控制器 | 短时掩盖脚步并制造可疑声源 | 暖色预警、铃声起音、实体控制器与 HUD 时机提示 |
| 医院 | 自动门/帘幕控制 | 短时扰乱视距并形成合法搜索岔路 | 冷色扫光、机械/布料声、门帘实体反馈 |
| 消防站 | 训练烟幕释放器 | 明显缩短视觉范围并轻度掩声 | 烟幕泛白、低频事件声、释放器状态灯 |
| 工厂 | 机器轰鸣控制台 | 强掩声、轻度视距扰动并生成工业声源 | 工业脉冲、机械层增强、控制台状态动画 |

所有效果使用确定性时间和连续包络，开始/结束不得跳变；最小视距倍率不得低于 `0.68`。事件只修改新产生的声音证据和合法视觉采样，不修改碰撞、抓捕距离或出口判定。四类机关的放置、交互、预警、激活、冷却与 AI 路线变化已由代表关卡 1/4/6/10 的运行态 QA 覆盖。

## 4. 动画与声音打磨

- `walk`、`run` 使用左右脚接触 marker；速度 cadence 仅在旧资产缺少 marker 时回退。
- 追捕者脚步只有在玩家合法可观察时才获得方向与宽泛远近分档，禁止从隐藏世界坐标计算声像。
- 正式 Foley 来自 Kenney CC0 素材；来源、许可证和运行时清单分别保存在：
  - `docs/licenses/KENNEY_AUDIO_CC0.md`
  - `public/audio/foley/manifest.json`
- 主题事件通过零坐标接口触发，避免音频层形成位置作弊。

## 5. 输入、暂停与 HUD

- 固定镜头、键盘、摇杆、路线箭头和左右声像共享同一屏幕坐标基。
- 手机摇杆使用 pointer capture；移出底盘仍能连续控制，抬手、取消、失去捕获、切后台或暂停都会清零。
- `Esc` 和顶部按钮打开暂停面板；暂停冻结模拟、关卡计时、追捕者和主题事件。
- 暂停面板提供继续、重开、静音、放大/缩小/重置视野。
- 完成首通后，平静状态减少持续藏柜文字卡；危险、无安全柜或近距离互动时恢复完整提示。
- 追逐态固定镜头朝向不随移动旋转；动态视野只调节距离与安全构图，键盘与摇杆的上下左右永远对应屏幕方向。
- 追捕者模型的世界渲染不受玩家 HUD 情报门控；只要处于合法镜头范围，追逐者必须持续可见。

## 6. 关卡与耐玩性

- 十关继续保持校园 3、医院 2、消防站 2、工厂 3 的主题结构。
- 每关使用独立精通目标组合；目标可来自无目击、双次脱身、空柜诱导、不搜柜、无藏身和限时逃生。
- 旧存档继续读取；采用新精通档案后，旧版金章不会直接冒充新档案金章。
- 标准与辅助结果分别保存最佳时间、精通与解锁权；辅助通关明确标记为不参与排名。
- Ghost 以输入变化关键帧记录，单局上限 `64 KiB`、本地库上限 `512 KiB`，带版本、关卡白名单与校验和；回放位置误差预算不超过 `0.1` 格。
- 平衡审计按新手、普通、熟练、速通四类策略生成确定性样本，检查策略顺序、相邻难度断崖和后期倒挂，并输出可直接调节的参数键。
- 难度验收采用结果门槛，不只检查参数递增：
  - 每关至少两处藏身点能覆盖 `0–250 ms` 反应延迟；
  - 无技巧直冲仍会失败；
  - 认证安全路线可完成；
  - 标准引导首通最多少量金章，避免评价系统失去长期目标。

## 7. 加载、纹理与渲染预算

- 场景级加载器：三并发、20 秒超时、最多三次尝试、指数退避和 jitter，关卡切换立即取消队列与 fetch。
- GLB 使用 `fetchArrayBuffer → parseAsync`，场景销毁时先取消网络，再释放动画、纹理、几何和 WebGL 资源。
- GLB 内声明的外链 PNG/KTX2 先被解析为依赖清单，再走同一加载器受控预取并映射给 Three.js；不得绕过超时与切关取消。
- KTX2 只在结构、动画、材质和高质量模式验证全部通过后替换运行时资产；Basis 解码器随站点发布。
- 首局角色使用独立 Bootstrap：主角 `1,573,988 B`、追捕者 `1,474,540 B`；结算警察按接近出口异步加载，为 `1,736,768 B`。节点、骨骼、材质与正式动作片段均与批准源模型保持合同一致。
- 四套主题 Bootstrap 分别为校园 `759,072 B`、医院 `567,388 B`、消防站 `639,596 B`、工厂 `819,840 B`。19 个环境 GLB 共用 BaseColor、Normal、ORM 三张 KTX2 内容寻址图集；独立环境资源从 `12,508,486 B` 降到 `6,594,136 B`，减少 `47.283%`，请求从 41 降到 21。
- 第一关完整首局关键资源为 `7,864,049 B`、16 个去重请求；按冷启动 `10 Mbps / 80 ms` 保守模型估算 `7.571 s`，同时通过 `8 MiB / 8 s` 总硬门槛。分类阈值为高精度入口与六段动画英雄柜体保留合理空间，但不能绕过总量和时间门槛。
- 量化静态网格做 CPU 世界矩阵合批前必须展开 `POSITION/NORMAL/TANGENT` 为 Float32，防止 Uint16 坐标被截断为三角撕裂。量化骨骼角色按 `geometry.boundingBox × matrixWorld` 的 GPU 解码路径校准绝对尺寸，禁止使用会重复应用解码变换的通用 SkinnedMesh 包围盒。
- 原始角色与 3D 美术母版全部保留；`kid-lod1.glb`、`villain-lod1.glb` 继续作为可复现构建与视觉对照输入，不进入首局请求图。
- 社交分享图使用独立 1200×630 高质量 JPEG，已接入 Open Graph 与 Twitter Card；不复用运行态低分辨率截图。
- 渲染质量档位同时约束像素比、动态灯、粒子、三角面、draw call、阴影面数和阴影提交。
- 静态环境阴影只在允许的质量档位启用；角色、交互柜和关键可读性资产不进入装饰性降级。
- `/models/`、正式 Foley、配乐和 Basis 解码器使用长缓存；内容更新必须同步提升 URL 版本或使用内容哈希文件名。

## 8. 回归与返工循环

每次候选版本依次执行：

1. `npm run typecheck`
2. `npm run lint`
3. `npm run build`
4. `node --experimental-strip-types --test tests/*.test.mjs`
5. 十关 guidance/difficulty 矩阵
6. 角色、柜门、主题包、KTX2、材质、骨骼与动画资产合同
7. 本地 HTTP 冒烟：HTML、GLB、KTX2、Basis、Foley、配乐均返回成功
8. 部署后鉴权冒烟、部署日志检查与平台截图复核

当前候选版本完整回归为 `279/279` 通过，并包含：

- 300 组藏点反应矩阵、十关直冲必败、十关认证藏身路线与十关引导生存验证；
- 主动主题机关、三证据上限、诱饵习惯化、真实岔路搜索与 AI 公平性；
- 同一 AI 帧的视觉+声音证据保留、诱饵点完整巡视完成事件与“调虎离山”运行态因果计分；
- 模型依赖可达性、内容哈希一致性、角色/主题 Bootstrap 视觉等价、KTX2 法线图集与六段柜门动画；
- Ghost 校验和、存储预算、跨帧率确定性和 `0.1` 格位置误差预算；
- 量化静态网格 Float32 合批与量化骨骼角色绝对尺寸防回归。

本地生产构建的运行态验收结果：

| 验收面 | 覆盖 | 结果 |
| --- | --- | --- |
| 十关环境美术 | 10 关全景；1/4/6/10 近景、追逐、躲藏与无 MultiDraw 回退 | 全通过，无资源/控制台诊断 |
| 追逐角色 | 代表主题双角色同屏，运行态世界高度校验 | 主角 `1.52 m`、追捕者 `1.88 m`，均在镜头内 |
| 性能 | 代表主题追逐态 120 帧采样 | P95 `8.5–9.9 ms`，P99 `9.8–10.3 ms`，`>33 ms` 比例为 0 |
| 柜体交互 | 桌面 10 关 + 390×844 移动端 10 关 | 对齐、进柜、柜内观察、开缝与出柜全部通过 |
| 主动机关 | 关卡 1/4/6/10 | 放置、交互、预警、激活、冷却与 AI 反应全部通过 |

发现以下任一情况必须返工并从第 1 步重新执行：

- 柜门与角色动作不同步；
- 角色/追捕者在合法可见状态消失；
- 屏幕方向与输入相反；
- 追捕者越过最后目击点流程或读取隐藏位置；
- 任一关无两条可生还藏身选择；
- KTX2 丢失节点、动画、材质、透明边缘或法线细节；
- 手机控件遮挡关键角色、出口或互动提示；
- 控制台报错、资源 404、加载无法取消或关卡切换泄漏。

最终交付以同一 Git 提交、同一构建归档和同一部署版本为准，禁止用未提交文件或不同 commit 的截图代替验收。

## 9. 本轮返工闭环

| 截图/回归发现 | 根因 | 修复 | 防回归 |
| --- | --- | --- | --- |
| 迷宫墙面出现斜向撕裂与碎三角 | gltfpack 量化 Uint16 顶点直接参与 CPU 世界矩阵烘焙，变换结果被截断 | 合批前将变换相关属性展开为 Float32，再应用矩阵 | 静态合同覆盖两条合批路径；MultiDraw 与 InstancedMesh 截图均复验 |
| 追逐态模型树显示可见，但画面几乎看不到人物 | SkinnedMesh 通用包围盒重复/错位应用量化节点解码，角色被缩成数厘米 | 改用几何边界乘世界矩阵的 GPU 等价路径，按批准身高统一校准 | QA 直接断言 `1.52 m / 1.88 m`，并要求两角色中心同时在视锥内 |
| 首局总量和时间达标但分类审计假红 | 旧导航/柜体分类阈值仍按早期低模原型制定 | 在不改变 `8 MiB / 8 s` 总硬门槛的前提下校准分类上限 | 生产态审计和超大未优化资产反例同时纳入测试 |

## 10. 发布验收结论

- 玩法：移动、追逐、失视到最后目击点、左右巡视、证据搜索、进出柜与主题机关形成完整闭环。
- 耐玩性：十关主题递进、分轨成绩、精通目标、Ghost 与平衡审计均已接线。
- 交互：固定朝向、动态视野、桌面/移动输入、暂停与柜内观察已回归。
- 美术：角色、迷宫、墙体、柜体、主题道具均使用正式 3D 资产；无几何线条占位回退。
- 沉浸：正式动作 marker、CC0 Foley、主题声景、空间遮挡和可见追捕者表演保持一致。
- 性能：首局预算、KTX2、请求去重、静态批处理、运行时质量档位和紧急降级均有硬门槛。
