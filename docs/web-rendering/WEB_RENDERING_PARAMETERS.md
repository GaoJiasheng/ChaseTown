# Three.js / WebGL 渲染参数参考

参考：

- `art-source/Concepts/00_art_direction_overview.png`
- `art-source/Concepts/04_school_environment_sheet.png`
- `docs/01_游戏设计文档_GDD.md` §8.3

本文件用于约束 Three.js 场景、灯光、色调映射与后处理。参数是视觉目标，不要求所有设备启用相同开销；移动端应按性能档位降级。

## 共用场景意图

- 学校走廊：偏冷、略低曝光，但角色轮廓和路线始终可辨。
- 警局出口：明显偏暖，作为安全方向提示。
- 警车灯：红蓝自发光可见，但不得让整屏泛光。
- 俯视 3/4 镜头的玩法可读性高于戏剧性暗部。

## `stylized`

| 项目 | 目标 |
|---|---|
| Tone mapping | `ACESFilmicToneMapping`，较低强度 |
| Exposure | `1.0` 至 `1.12` |
| Contrast / Saturation | 轻度增强，可由后处理 LUT 或 CSS 状态层实现 |
| 主光 | 暖色半球光 + 柔和方向光 |
| AO | 中低强度；低性能设备可关闭 |
| Bloom | `0.15` 至 `0.35`，只覆盖灯具和自发光 |
| Vignette | 常态很弱；危险状态使用独立红色覆盖层 |

视觉目标：材质仍有真实质感，阴影比写实风柔和，整体温暖且适合移动端阅读。

## `photoreal`

| 项目 | 目标 |
|---|---|
| Tone mapping | `ACESFilmicToneMapping` |
| Exposure | `0.9` 至 `1.05` |
| Contrast / Saturation | 对比更强，饱和度保持中性 |
| 主光 | 方向光软阴影，学校区域略偏冷 |
| AO | 中高强度，但不得产生脏边或黑角 |
| Bloom | `0.08` 至 `0.2`，高阈值 |
| Film grain | `0.08` 至 `0.16`；低性能设备关闭 |
| Depth of field | 游戏中关闭，仅允许胜负定格画面使用 |

视觉目标：强调接触阴影和材质层次；警局暖色与走廊冷色形成对照，反派暗部不能黑成剪影。

## `blind-box`

| 项目 | 目标 |
|---|---|
| Tone mapping | 中性或 ACES 低强度 |
| Exposure | `1.05` 至 `1.18` |
| Contrast / Saturation | 低对比、较高饱和度 |
| 主光 | 大面积柔光，突出圆润轮廓 |
| AO | 低强度、短半径 |
| Bloom | `0.25` 至 `0.5`，服务于塑料高光 |
| Vignette / Grain | 默认关闭 |

视觉目标：塑料和搪胶高光干净但不过曝；背景克制，让圆润角色突出，同时保留迷宫辨识度。

## 危险反馈层

危险反馈不属于基础风格，由游戏状态独立驱动：

| 项目 | 目标 |
|---|---|
| 颜色 | 深红，约 `#B00018` |
| 强度 | 距离较远时 `0`，濒临抓捕时最高约 `0.45` |
| 过渡 | 平滑插值，避免闪烁 |
| 节奏 | 可与心跳音效同步，不得遮挡 HUD |

## 性能降级顺序

1. 降低设备像素比上限。
2. 降低阴影贴图分辨率和更新频率。
3. 关闭 AO、颗粒与 Bloom。
4. 减少动态灯光，只保留主方向光和半球光。
5. 保持角色、出口、追捕警示与路径标识，不以可玩性换画质。

## 验收

- 三套预设均保留警局出口暖色提示和警车红蓝灯。
- UI、触控区域和安全区不受后处理影响。
- Chrome、Safari 桌面端可正常运行；移动端在降级后仍可操作。
- 切换预设不得重新下载整套模型，也不得泄漏材质、纹理或 WebGL 资源。
