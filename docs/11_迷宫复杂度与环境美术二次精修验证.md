# Web 版迷宫复杂度与环境美术二次精修验证

更新日期：2026-07-23
状态：本地与浏览器技术门禁 `PASS`；主观体验仍以用户试玩验收为准

## 1. 本轮目标与结论

本轮针对“迷宫过于简单”和“环境 3D 仍有积木感”做了两条独立收口：

- 迷宫：封闭原有长直捷径，用多转折 dogleg、晚段分支、回环和视线风险点重构十关拓扑；不以单纯增加面积伪造难度。
- 美术：补齐环境 BaseColor / Normal / ORM，新增四主题连续四米墙段、路口地标、墙脚/道具接触阴影和主题房间簇；固定镜头改为更能显示纵深的三分之四方位。

最终十关均保持单一连通空间；最短路线盲冲 10/10 会被抓，认证藏柜路线 10/10 可逃脱。原版浏览器美术 QA 完成十关主路径、四主题近/远/藏匿镜头和四主题无 MultiDraw 回退，共 10 + 12 + 4 组运行态采证，控制台、网络、HTTP 与运行时诊断为 0。

## 3. 环境 3D 二次精修

### 3.1 连续建筑模块

每个主题新增 `ArchitectureWallWide` 四米连续墙，不是把两块旧墙按 X 轴拉伸：

- 校园：连续护墙板、阅览室冠梁、三组展窗与黄铜徽章。
- 医院：连续踢脚/导视带/扶手、弧形顶梁与观察窗。
- 消防站：连续砖墙、钢制基座、警示带、桁架与消防设施面板。
- 工厂：连续波纹钢、下部防撞板、双警示带、锯齿冠梁与设备面板。

运行时识别同一边界上相邻的两米墙段，自动替换为四米模块；端墙、门洞和转角不参与合并，避免覆盖交互入口。低端无 MultiDraw 路径保留四米正式模块，仅把少量无法配对的 B/C 单格余段合并到 A 材质批次。

新增模型合同要求：

- 四米模块宽度 3.9–4.2 m、合理墙深与墙高；
- 至少 400 triangles，并声明 `chasing_continuous_span_meters = 4`；
- A/B/C 两米墙必须保持不同天际线；
- 路口 `ArchitectureJunction` 必须有连续曲线轮廓、净空和明确的玩法地标语义。

### 3.2 PBR 与接触关系

- 新增可重复生成的 512×512 glTF ORM：R=AO、G=Roughness、B=Metallic。
- 有 BaseColor + Normal 的环境材质必须同时有 MetallicRoughness 和 AO，且二者复用同一个紧凑 ORM texture index。
- 墙脚与大型道具各使用一个 InstancedMesh 接触阴影批次；不以全屏重型 GTAO 掩盖材质问题。
- 降低环境填充、半球光和反向补光，保留 ACES、主方向阴影与主题色温。
- 从地图边界 flood-fill 标记外部 void；只给含叙事道具的封闭非行走区铺满主题房间地面，同一房间去重。大房间使用平静的 secondary 材质，强警示 service 材质仅用于不超过 12 格的小工具间，不向墙外生成孤立平台。

### 3.3 镜头可读性

- 镜头世界方位仍固定，输入与同一 immutable basis 投影，因此转向后上下左右不会反。
- 横向方位增加三分之四偏角，俯角从约 55° 降到约 47°，显示墙帽、立面和道具纵深。
- 探索状态保留原有大视野；进柜、出柜、窥视与胜负演出才动态靠近。用户手动近景不再被旧的 13.5 m 下限截断，最低安全距离为 11.6 m；追逐双人安全构图仍可强制拉远。
- 地图边缘的藏柜演出会把焦点最多向迷宫内部平移两格，并用 FOV / aspect / distance 投影门限保护角色画幅；相机方位与输入基准不旋转，中心藏柜、自由移动及双角色追逐不受影响，竖屏会自动缩小偏移。

## 6. 截图返工循环记录

1. 首轮截图发现每两米立柱与铭牌形成高频 wallpaper、非行走区道具缺乏承托；新增连续四米墙与主题房间地面。
2. 首轮严格 QA 发现完整 ORM 使 renderer textures 从旧门槛 224 增至 248；没有隐藏该事实，而是确认 live scene textures 仍 ≤72，并将完整 PBR 的硬上限明确设为 256。
3. 首轮 fallback 发现校园/医院 531/535 calls；连续墙合并与余段批次收敛后降为 495/503，消防/工厂最终为 508/447。
4. 第 10 关拓扑变化使旧 chase 截图坐标失效；更新为新迷宫中有 LOS 的 `(player 9,4 / chaser 5,4)` 并恢复追逐性能采样。
5. 曾尝试把背景房间扩大到 5×5；截图出现地面伸出墙外的“大平台”，判定不合格。固定 3×3 回退仍会在地图边缘留下孤岛，因此最终改为“边界 void 排除 + 封闭房间连通域整体铺装”，既不外伸也不留道具脚下的孤砖。
6. L4/L10 藏匿镜头曾让空背景占据半屏以上；最终仅在藏柜演出时将焦点向地图内部投影安全地平移，角色仍在安全画幅，固定操作方向不变。
7. 模型重建/HMR 期间曾记录旧缓存缺少四米墙和并行 QA `ready=false` 的瞬时错误；所有改动停止后用原版脚本从零重跑十关、四主题追逐与四组 fallback，最终 `diagnostics=[]`。
8. 最新运行截图复核发现宽镜头下敌我轮廓仍不够清楚、地图边缘空地偏多、柜内观察过暗；默认距离收紧为 14.8，外缘自由移动加入投影安全的场内偏移，正式角色材质增加仍受深度遮挡的蓝/红 Fresnel 轮廓，柜内观察缝与曝光只在按住观察时渐进增强。
9. 严格浏览器 QA 复现首关 `78` 个 shader programs 与 `82` 个 live scene textures。根因不是跨关泄漏，而是 ready 前零动态灯首帧和装饰灯加入后重复编译整套 PBR 变体，以及装饰模型在纹理去重前被克隆进场景。修复后十关为 `46–49 / 45–61`；可选装饰组装失败时还会保护正在使用的 canonical textures。
10. 最终再次跑通十关主视图、四主题近/远/追逐/藏匿、四主题无 MultiDraw 回退、十关柜子进入/隐藏/退出循环及完整窥视动作；浏览器 exception、warning/error log 和 invalid scene textures 均为 `0`。

最终结论限定为“自动化、资产合同和当前桌面浏览器运行态通过”。美术主观满意度仍以用户实际试玩为准；不能用自动化 `PASS` 代替主观验收。
