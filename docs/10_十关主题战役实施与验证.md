# Web 版十关主题战役实施与验证

更新日期：2026-07-22

## 本次交付边界

本轮把单一校园原型扩展为 10 关顺序战役，并同步完成环境美术、节奏、关卡公平性和浏览器运行质量收口。运行时仍为 React + Three.js + WebGL，不依赖 Unity。

| 章节 | 关卡 | 核心环境语言 |
| --- | ---: | --- |
| 校园 | 1–3 | 护墙板、蓝色立柱、木地板、奖杯柜、饮水机、自行车架、导视 |
| 医院 | 4–5 | 洁净墙板、扶手、呼叫面板、病床、急救车、输液架、轮椅、隔帘 |
| 消防站 | 6–7 | 砖/钢墙、红色管线、反光警示、消防车、装备架、卷盘、消防栓 |
| 工厂 | 8–10 | 波纹钢、配电箱、管线、储罐、控制台、输送带、护栏、料箱 |

关卡按顺序解锁，浏览器本地保存解锁进度和最佳时间。每关拥有独立路线、巡逻点、藏身点、主题配色、雾效、地标和难度参数。

## 环境美术实现

- 四套主题包由 `scripts/build-theme-kits.py` 可重复生成，源母版保留在 `art-source/Environment/ThemeKits/`。
- 主题包不是运行时方盒替代物；每套 GLB 包含命名建筑墙体和 5–6 个可独立摆放的英雄道具。
- 墙体使用真实的护墙板、扶手、砖块、管线、压纹钢、踢脚与设备面板层级；重复墙段经过减面后为 576–1188 triangles/bay。
- 复用项目 512px PBR 彩色与 OpenGL 法线库，并以 WebP 内嵌到主题包。单包 1.17–1.87 MiB，总计约 6.1 MiB。
- 运行时按材质合并静态网格、按墙体批次实例化，并把重复贴图引用归一到共享 GPU Texture；代表关卡实际场景纹理引用为 59–65。
- 入口区域增加主题英雄道具组和地面导向标识，避免大面积无内容空地；主题道具参与真实阴影和相机遮挡。

## 公平性和可见性

- 相机世界方向固定，WASD、方向键和触控按钮共享同一个屏幕坐标转换，角色转身不会改变按键含义。
- 玩家暴露时，追捕者 3D 模型始终保留在世界中；“HUD 不知道坏人位置”不会再关闭模型。
- 第一关 7 个实体碰撞点与实际 bench/tree/shrub/car/hoop/desk/podium 模型严格同坐标，不存在隐形碰撞。
- 关卡 2–10 的 28 个 `visionOnlyBlockers` 都渲染为可穿越的主题烟尘、消毒蒸汽或工业蒸汽；玩家看得见 AI 为何断视线。
- 进柜、隐藏、窥视、出柜与追捕者查柜继续使用正式骨骼动画和带门轴的储物柜 GLB，模拟时长、角色动作和柜门动作同步。

## 自动化与浏览器验收

逻辑回归覆盖：

- 10/10 关最短路线盲冲都会触发警觉、追逐并被抓。
- 10/10 关认证藏柜路线都能断开视线、完成隐藏并到达出口。
- 20% 速度、固定步长、固定镜头方向、感知公平性、角色/柜门动作合约均有自动测试。

最终发布门禁：

