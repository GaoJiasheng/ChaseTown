# Codex 外包资产规格书

> 版本 v0.3 · 2026-07-15 · 面向 Codex 的明确、可验收的 Web 3D 资产制作要求
> 运行时：**Three.js + WebGL 2** · 交付格式：**glTF 2.0 Binary（GLB）+ PBR 贴图** · 单位：米
> v0.3 变更：项目改为纯 Web；制作源统一进入 `art-source/`，浏览器资产统一进入 `public/models/`。

---

## 0. 最高原则（所有资产必须遵守）

1. **真实图像质感优先**：所有材质使用有真实细节的 PBR 纹理；严禁用纯色、几何线条、色块或图形叠加充当“模型”或“贴图”。
2. **一套骨骼贯穿三风格**：同一角色的三套风格共用同一骨骼层级、命名和动画语义，保证浏览器运行时无缝切换。
3. **Web-ready**：最终 GLB 由 Three.js `GLTFLoader` 直接加载，无需在代码里临时修缩放、朝向、轴心、材质通道或骨骼。
4. **移动浏览器友好**：面数、材质数、纹理分辨率、下载体积和 GPU 内存遵守预算；写实风格必须有 LOD。
5. **源与运行时分离**：`.blend`、高模、原始贴图和交换格式只放 `art-source/`；应用只加载 `public/models/`。
6. **可复核**：每个资产附预览图、线框图、面数/贴图/材质/动画/体积报告，并在真实浏览器中逐条验收。

## 1. 通用技术规范

### 1.1 运行时 GLB

| 项 | 要求 |
|---|---|
| 格式 | glTF 2.0 Binary `.glb`；浏览器运行时不直接加载制作源或交换文件 |
| 单位 / 缩放 | 1 unit = 1 米；导出前 Apply Transform；加载后对象 scale 必须为 `(1,1,1)` |
| 坐标朝向 | 右手坐标，Y-up；角色面朝 +Z；脚底贴 Y=0；轴心在双脚中心地面 |
| 拓扑 | 变形区以四边面拓扑为主；无坏面、非流形、翻转法线或意外重复几何 |
| UV | 单角色 1–2 套 UV；充分利用 0–1；除明确复用外无重叠；不使用 UDIM 作为浏览器运行时依赖 |
| 切线 | Normal Map 资产必须导出正确 tangent，避免不同设备上法线表现漂移 |
| 材质 | glTF PBR Metallic-Roughness；只使用 Three.js/GLTFLoader 明确支持的扩展 |
| 纹理 | 运行时 PNG/JPEG/WebP 或 KTX2；颜色纹理为 sRGB，数据纹理保持线性 |
| 命名 | 节点、mesh、material、skin、animation clip 使用稳定英文名称，不依赖自动生成序号 |
| 外部依赖 | GLB 优先自包含；若纹理外置，必须使用 `public/models/` 内相对 URL，并在报告中列清单 |
| 交付附件 | 三视角预览、线框图、面数/材质/贴图/动画/文件体积报告、浏览器验收截图 |

制作源可以包含 `.blend`、高模、原始 PNG/TGA 和必要的 `.fbx` 交换文件，但它们只能放在 `art-source/`；最终接入标准始终是 GLB。

### 1.2 glTF PBR 通道

| 纹理 | glTF / Three.js 约定 |
|---|---|
| Base Color | sRGB；RGB 为颜色，A 可作透明度；除头发卡片等必要对象外避免透明 |
| Normal | 线性、OpenGL Y+；必须包含真实细节，不能是纯 `(128,128,255)` |
| Metallic-Roughness | 线性；**G=Roughness、B=Metallic**，未用通道按 glTF 默认值处理 |
| Occlusion | 线性；R=AO；需要时与 Metallic-Roughness 共用图像，但纹理槽语义仍要正确 |
| Emissive | sRGB；警灯、顶灯、指示牌使用；强度由材质配置控制 |

- 禁止把其他引擎的通道打包约定原样塞入 GLB。
- 单角色不超过 2 个主要材质；环境单道具不超过 1–2 个材质。
- 尽量复用材质与纹理，避免为颜色微调复制整套贴图。
- KTX2/Basis 压缩版必须与未压缩基准图目视一致，不能出现明显色带、法线破损或透明边缘。

### 1.3 浏览器加载与体积

- 使用 glTF Validator 检查错误；Error 必须为 0，Warning 需逐条解释。
- 使用项目实际的 `GLTFLoader` 与解码器加载，不以 DCC 预览成功代替浏览器成功。
- 资源 URL 不得 404；GLB、外置纹理和解码器 MIME 类型正确。
- 模型加载完成后首帧必须可见，不能落在相机外、地面下或因材质/灯光全黑。
- 每个 GLB 报告原始体积、压缩体积、纹理占比和首次加载耗时。

## 2. 统一骨骼与动画规范（三风格共用）

### 2.1 骨骼

- 标准人形层级：Hips / Spine / Chest / Neck / Head / 双臂（Shoulder→Hand）/ 双腿（UpperLeg→Toes）。
- 同一角色三套风格的 joint 层级、命名、bind pose 和朝向一致。
- 无面部骨也可；Demo 阶段表情可使用 morph target，非必需。
- T-pose 或 A-pose 绑定；导出后在 Three.js 中检查 skin、inverse bind matrix 和骨骼权重。
- 单顶点权重数量控制在浏览器友好范围；无孤立骨、零权重顶点和明显拉伸。

### 2.2 动画集

| Clip | 用途 | 循环 | 备注 |
|---|---|---|---|
| `Idle` | 待机 | 是 | 轻微呼吸/张望 |
| `Run` | 主要移动 | 是 | 面朝 +Z 原地跑 |
| `Walk` | 慢速/巡逻 | 是 | 供坏人 Patrol |
| `TurnLeft` / `TurnRight` | 急转身 | 否 | 可选增强 |
| `LookAround` | 坏人搜索 | 是 | 警觉感 |
| `Scared` / `Caught` | 小孩被抓 | 否 | 失败演出 |
| `Celebrate` | 小孩获救 | 否 | 胜利演出 |
| `Point` / `Alert` | 警察演出 | 否 | 指认/对讲机 |

- 30 fps 烘焙；Root Motion 关闭，水平位移由游戏逻辑和导航控制。
- clip 名称在 GLB 内可直接枚举，起止帧无额外静止帧。
- 循环 clip 首尾姿态和速度连续，无抖动、滑步、穿模或脚底明显漂移。
- 使用 Three.js `AnimationMixer` 在三套真实角色 GLB 上逐一播放；只在空骨架上成功不算验收。
- 角色体型不同时可以调整动画，但不得破坏共用层级和运行时切换契约。

## 3. 角色资产——小孩（Kid）

> 参考：`art-source/Concepts/01_kid_character_sheet.png`

通用设定：8–10 岁儿童，身高约 1.3 米，背双肩书包，灵动、无辜、易共情。

| 风格 | 面数预算 | 贴图 | 材质要点 | LOD |
|---|---|---|---|---|
| 写实卡通 | 18k–30k tris | 2K | 真实布料/皮肤，头略大，眼睛有神 | LOD0/1 可选 |
| 完全写实 | 40k–70k tris（LOD0） | 4K 源图，运行时按设备压缩 | 拟真皮肤、织物法线、头发卡片 | **必须 LOD0/1/2** |
| 盲盒潮玩 | 12k–20k tris | 2K | 光滑塑质、圆润无锐边、干净高光、大眼 | LOD0 |

- 服装为连帽卫衣/校服 + 双肩书包；书包是远距离识别符号。
- 动作气质轻盈、敏捷。
- 拒绝几何拼贴假人、色块脸、无真实材质细节的塑料壳。

## 4. 角色资产——坏人（Villain / Chaser）

> 参考：`art-source/Concepts/02_villain_character_sheet.png`

通用设定：成年男性，约 1.85 米，高大、有压迫感，深色连帽外套/风衣，不血腥、不持械。

| 风格 | 面数预算 | 贴图 | 材质要点 | LOD |
|---|---|---|---|---|
| 写实卡通 | 20k–32k tris | 2K | 深色皮革/布料、帽檐阴影、魁梧比例 | LOD0/1 可选 |
| 完全写实 | 45k–75k tris（LOD0） | 4K 源图，运行时按设备压缩 | 风衣褶皱、皮肤/胡茬、脏旧质感 | **必须 LOD0/1/2** |
| 盲盒潮玩 | 14k–22k tris | 2K | 圆润但保留大手大肩，深色塑质 | LOD0 |

- 剪影必须与小孩形成“大/小、暗/亮”的强对比。
- 帽檐或围巾可遮挡部分面部，保持神秘威胁感并降低非关键面部成本。
- 拒绝僵尸化、血腥或武器化处理，维持“危险陌生人”的大众尺度。

## 6. 环境资产——学校迷宫

> 参考：`art-source/Concepts/04_school_environment_sheet.png`

场景使用模块化套件组成，但必须有真实材质和建模细节，不能把灰盒直接当成正式资产。重复模块要能通过共享 geometry/material 或 `InstancedMesh` 降低 Draw Call。

### 6.1 模块化建筑套件（2 米网格）

| 资产 | 规格要点 | 贴图 |
|---|---|---|
| Wall 直段/转角/端头 | 2 米模数，接缝无错位，边缘有倒角和结构细节 | 真实砖/漆面 2K、Trim 图集 |
| Floor 走廊/教室/操场/草坪 | 无缝平铺，避免 Z-fighting | BaseColor/Normal/ORM |
| Door 前门/后门/教室门 | 前后门是地标，轮廓醒目 | 木/金属/玻璃 PBR |
| Ceiling / Light | 顶灯 Emission 辅助俯视照明 | 发光贴图或材质参数 |

### 6.2 核心道具

- 走廊：储物柜、公告栏、灭火器、垃圾桶、天花灯。
- 教室：课桌椅、黑板、讲台、散落书本/书包。
- 操场：篮球架、跑道线、长椅、树木、草丛、路灯。
- 警察局门口：警车、独立红蓝警灯 Emission、门厅暖光、指示牌。
- 叙事道具：翻倒椅子、掉落书包。

### 6.3 环境预算

| 项 | 目标 |
|---|---|
| 单道具面数 | 简单件 <1k tris；复杂件如警车 <8k tris |
| 贴图 | 复用图集/Trim Sheet；避免每个小物独占 2K 贴图 |
| 平铺材质 | 墙地面使用 tileable 纹理，不能逐面唯一贴图 |
| LOD | 树、警车、警局外观等大件提供 LOD |
| 重复实例 | 模块和重复道具必须支持共享材质/几何或实例化 |

### 6.4 关卡交付形式

- 每个模块/道具交付独立 GLB，并提供稳定资源 ID。
- 提供一份已拼好的学校迷宫 GLB 或模块实例清单，以及对应 `level.json`。
- `level.json` 至少记录模块实例 Transform、起点、终点、巡逻点、碰撞范围和可行走网格/导航图。
- 碰撞与导航数据必须与可见几何一致；无穿墙、卡死、不可达终点或死胡同寻路漏洞。
- 关卡在 Three.js 页面中加载后，桌面与手机相机都能读清前门、警察局和主要路径。

## 7. 验收清单（每个资产逐条对照）

### 通用

- [ ] GLB 通过 glTF Validator，Error=0；Warning 已解释。
- [ ] `GLTFLoader` 加载无异常，浏览器控制台无错误、资源请求无 404。
- [ ] 加载后 scale=1、角色面朝 +Z、轴心与脚底位置正确，不靠运行时代码补偿。
- [ ] BaseColor、Normal、Metallic-Roughness、AO、Emission 通道语义正确。
- [ ] Three.js/WebGL 光照下无全黑、过曝、透明排序灾难或明显法线错误。
- [ ] 面数、纹理、材质、动画和文件体积在预算内，附机器可读报告。
- [ ] 全 PBR 真实质感，无几何色块/线条充数。
- [ ] 桌面 Chrome/Safari 和手机 Safari/Chrome 至少各一组验收截图。

### 角色

- [ ] skin、joint 与 inverse bind matrix 正常，无零权重顶点。
- [ ] 三风格共用骨骼层级/命名，可复用同一动画语义。
- [ ] `AnimationMixer` 可播放完整 clip，30 fps、Root Motion 关闭，无滑步/穿模。
- [ ] 小孩、坏人和警察在实际俯视相机与手机尺寸下仍清晰可辨。
- [ ] 写实风格 LOD 齐全，切换无明显跳变或动画重置。

### 环境

- [ ] 模块按 2 米网格拼接，无缝、无 Z-fighting。
- [ ] 平铺材质无接缝，重复对象可实例化。
- [ ] 可行走网格/导航图覆盖正确，起点到终点可达。
- [ ] 警车 Emission、警察局暖光和关键地标在 WebGL 中清楚可见。

## 9. 交付目录约定

```text
art-source/
├─ Concepts/                         定稿概念图与返工参考
├─ Characters/                       Kid / Villain / Police 当前静态与绑定 Blender 母版
├─ Environment/                      ModularKit / Props / SampleMaze 正式 FBX
├─ _Shared/Animations/               共用骨骼与动画制作源
└─ _Source/                          第三方许可证、来源链接与清单

历史候选、拒稿、`.blend1`、重复导出、批量预览、线框和评审缓存不进入仓库。制作过程可在本地工作目录保留；Git 只接受当前母版、运行依赖和必要来源记录。

public/models/
├─ characters/                       角色运行时 GLB；风格变体由 manifest 映射
├─ environment/                      模块与道具运行时 GLB
└─ SharedTextures/                   必须外置时使用的运行时共享纹理

tools/art_pipeline/                   导出、压缩、验证、报告脚本
```

每个正式交付必须同步更新当前制作母版和运行时 GLB，并通过自动化资产门禁。应用资源 manifest 只引用 `public/models/...`；不得把 `art-source/` URL、绝对本地路径或临时下载目录写入产品代码。
